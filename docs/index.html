Vexium Documentation

The complete reference for the Vexium programming language — a human-readable, English-like compiled language that makes coding as natural as describing your ideas.

English-like Syntax Compiled to Native Cross-Platform v2.0
Quick Glance

Vexium uses let ... be for variables, fn for functions, struct for objects, and English phrases like is greater than for comparisons. No semicolons, no curly braces for blocks — just clean, indented code.

vexlet name be "World"
display "Hello, {name}!"

fn greet(person):
    display "Welcome, {person}!"

greet("Vexium Developer")

Installation

Vexium compiles to a single native executable. Download the latest release for your platform from GitHub Releases.

Build from Source

Requires a C compiler (GCC, Clang, or MSVC).

bash# Clone the repository
git clone https://github.com/RockyOmvi/Vexium.git
cd Vexium

# Build with make
make

# Or compile directly
gcc -O2 -o vexium src/*.c -lm

Hello, World!

Create a file called hello.vxm:

vexdisplay "Hello, World!"

Run it:

bashvexium run hello.vxm
outputHello, World!

Variables & Constants

Variables are declared with let ... be. Constants use const ... be and cannot be reassigned.

Variables

vexlet name be "Alice"
let age be 25
let score be 98.5
let active be true

# Reassignment (no 'let' keyword)
age be 26

Constants

vexconst PI be 3.14159
const APP_NAME be "VexApp"

# PI be 3.0   ← This would throw an error!
Note

Reassigning a constant produces a runtime error: Cannot reassign constant 'PI'

Data Types

Vexium is dynamically typed. Every value has one of these types:

Truthiness

These values are falsy — everything else is truthy:

  • false
  • nothing
  • 0 (int) and 0.0 (float)
  • "" (empty string)
  • [] (empty array)

Strings

Strings in Vexium support interpolation, concatenation, and repetition.

String Interpolation

Embed variables directly in strings using curly braces {variable}:

vexlet name be "Vexium"
let version be 2

display "Welcome to {name} v{version}!"
# Output: Welcome to Vexium v2!

You can also interpolate field accesses:

vexdisplay "{dog.name} is {dog.age} years old"

Concatenation & Repetition

vexlet greeting be "Hello" + " " + "World"
display greeting     # Hello World

display "ha" * 3     # hahaha

String Indexing

vexlet text be "Vexium"
display text[0]      # V
display text[-1]     # m
display len(text)    # 6

Display & Input

display

display is Vexium's built-in print statement. It outputs a value followed by a newline.

vexdisplay "Hello, World!"
display 42
display [1, 2, 3]
display true
display nothing

input()

input() reads a line of text from standard input. Pass an optional prompt string.

vexlet name be input("What is your name? ")
display "Hello, {name}!"

let age be int(input("Enter your age: "))
display "You are {age} years old"

Type Casting

Convert between types using built-in cast functions:

vexdisplay int("42")       # 42
display int(3.14)       # 3
display float(10)       # 10
display str(3.14)       # "3.14"
display type(42)        # int
display type("hello")   # str
display type([1,2])     # array

If / Elif / Else

Conditional branching uses if, elif, and else followed by a colon. The body is indented.

vexlet score be 85

if score is greater than 90:
    display "Grade: A"
elif score is greater than 80:
    display "Grade: B"
elif score is greater than 70:
    display "Grade: C"
else:
    display "Grade: F"

Execute a block repeatedly while a condition is true.

vexlet count be 0
while count is less than 5:

For Loops

Vexium supports two types of for loops: range-based and for-each.

For Range

Iterate over a numeric range with for i in start to end:

vexfor i in 1 to 5:
    display i
# Output: 1, 2, 3, 4
Note

The end value is exclusive — for i in 1 to 5 iterates 1, 2, 3, 4.

With Step

Use by to specify a step size:

vexfor i in 0 to 10 by 2:
    display i
# Output: 0, 2, 4, 6, 8

For Each

Iterate over arrays and other iterables:

vexlet fruits be ["apple", "banana", "cherry"]

for each fruit in fruits:
    display fruit

Repeat

Execute a block a fixed number of times:

vexrepeat 3 times:
    display "Vex!"
outputVex!
Vex!
Vex!

Match

Pattern matching with match and => arms:

vexlet day be 3

match day:
    1 => display "Monday"
    2 => display "Tuesday"
    3 => display "Wednesday"
    4 => display "Thursday"
    5 => display "Friday"

Match works with strings too:

vexlet command be "start"

match command:
    "start" => display "Starting..."
    "stop"  => display "Stopping..."
    "help"  => display "Showing help"

Break & Skip

Use break to exit a loop early, and skip (Vexium's continue) to jump to the next iteration.

vexfor i in 0 to 10:
    if i is 5:
        break
    display i
# Output: 0, 1, 2, 3, 4

for i in 0 to 5:
    if i is 2:
        skip
    display i
# Output: 0, 1, 3, 4

Defining Functions

Functions are declared with the fn keyword:

vexfn greet(name):
    display "Hello, {name}!"

greet("Alice")
greet("Bob")

Multiple Parameters

vexfn add(a, b):
    give back a + b

let result be add(10, 20)
display result   # 30

Pass (Empty Body)

Use pass as a placeholder for an empty function body:

vexfn todo():
    pass

Return Values

Use give back to return a value from a function:

vexfn square(x):
    give back x * x

fn max_of(a, b):
    if a is greater than b:
        give back a
    give back b

display square(7)          # 49
display max_of(10, 25)    # 25

To return nothing explicitly:

vexfn maybe_greet(name):
    if name is "":
        give back nothing
    display "Hi, {name}!"

Recursion

Functions can call themselves recursively:

vexfn factorial(n):
    if n is at most 1:
        give back 1
    give back n * factorial(n - 1)

display factorial(5)    # 120
display factorial(10)   # 3628800
vexfn fib(n):
    if n is at most 1:
        give back n
    give back fib(n - 1) + fib(n - 2)

display fib(10)    # 55

Arrays

Arrays are ordered, mutable collections of any type.

vexlet numbers be [1, 2, 3, 4, 5]
let mixed be [42, "hello", true, 3.14]
let empty be []

Indexing

vexlet colors be ["red", "green", "blue"]
display colors[0]     # red
display colors[-1]    # blue (last element)
display len(colors)   # 3

Push & Pop

vexlet items be [1, 2, 3]
push(items, 4)       # [1, 2, 3, 4]
let last be pop(items) # 4, items is now [1, 2, 3]

Nested Arrays

vexlet matrix be [[1, 2], [3, 4], [5, 6]]
display matrix[0][1]  # 2
display matrix[2][0]  # 5

Maps (Dictionaries)

Maps store key-value pairs. Keys must be strings.

vexlet person be {"name": "Alice", "age": 30, "active": true}

display person["name"]    # Alice
display person["age"]     # 30
display len(person)       # 3
Warning

Map literals must be written on a single line. Multi-line map definitions may produce unexpected indentation tokens.

Structs

Structs define custom data types with fields and methods — Vexium's object-oriented programming model.

vexstruct Dog:
    has name: str
    has age: int

    can bark(self):
        display "{self.name} says: Woof!"

    can info(self):
        display "{self.name} is {self.age} years old"

Fields

Declare fields with has fieldname: type. The type annotation is informational — Vexium is dynamically typed.

  • has name: str — a string field
  • has age: int — an integer field
  • has score: float — a float field

Methods

Methods are defined inside a struct with can method_name(self, ...). The first parameter must be self to access the instance.

vexstruct Counter:
    has value: int

    can increment(self):
        self.value be self.value + 1

    can get_value(self):
        give back self.value

    can reset(self):
        self.value be 0

let c be Counter(0)
c.increment()
c.increment()
c.increment()
display c.get_value()   # 3

Constructors

Structs are instantiated by calling the struct name as a function. Arguments are assigned to fields in declaration order.

vexstruct Point:
    has x: int
    has y: int

let p be Point(10, 20)
display p.x    # 10
display p.y    # 20

Complete OOP Example

vexstruct BankAccount:
    has owner: str
    has balance: float

    can deposit(self, amount):
        self.balance be self.balance + amount
        display "Deposited {amount}. Balance: {self.balance}"

    can withdraw(self, amount):
        if amount is greater than self.balance:
            display "Insufficient funds!"
            give back false
        self.balance be self.balance - amount
        display "Withdrew {amount}. Balance: {self.balance}"
        give back true

let account be BankAccount("Alice", 1000.0)
account.deposit(500.0)
account.withdraw(200.0)

Error Handling

Use attempt / otherwise for error handling (Vexium's try/catch):

vexattempt:
    display "Trying something risky..."
    let result be 10 / 0
otherwise err:
    display "Caught error: {err}"
Note

The err variable after otherwise captures the error message as a string.
